The `System` endpoints in R2R provide operational insights and utilities for monitoring and maintaining the platform's core health. These endpoints are primarily used to verify system availability, perform diagnostics, and ensure the infrastructure is responsive for API consumers and internal services.

## Available Endpoints

| Method | Endpoint                      | Description         |
| ------ | ----------------------------- | ------------------- |
| GET    | [/health](/reference/health-check) | Check system health |
